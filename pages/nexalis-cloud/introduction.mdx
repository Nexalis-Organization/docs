## Two Ways to Access Your Data

Nexalis Cloud offers two complementary data access methods, each optimized for different use cases:

### 1. Real-Time API

A high-performance **HTTP API** for querying time-series data in real time. Powered by [Warp10](https://www.warp10.io/), this API is designed for:

- **Real-time applications** requiring sub-second response times
- **Custom time-range queries** with advanced filtering and aggregations
- **Edge analytics** and operational dashboards
- **Event-driven architectures** and alerting systems

Perfect for applications that need the latest sensor readings or want to query historical data with precise time windows.

### 2. Lakehouse Sharing

**Read-only access to your data** through native lakehouse sharing protocols. Nexalis deploys your data on the lakehouse platform and cloud region that matches your infrastructure â€” whether that's **Databricks**, **Snowflake**, or **Microsoft Fabric** on **AWS**, **Azure**, or **GCP**. You connect using native shortcuts from your own lakehouse platform:

#### Three Native Connection Options

- **Databricks-to-Databricks** â€” Delta Sharing protocol for read-only table access
- **Fabric-to-Fabric** â€” OneLake shortcuts and external data shares
- **Snowflake-to-Snowflake** â€” Native data sharing with secure views

Nexalis matches your specific combination of cloud provider, lakehouse platform, and region to ensure seamless, cost-effective data sharing with zero egress fees when in the same region.

#### Benefits

- **Zero data replication** â€” Data stays in Nexalis lakehouse, you query it directly
- **Native integration** â€” Works seamlessly with your existing lakehouse platform
- **Optimized deployment** â€” Data hosted in the same cloud provider and region as your infrastructure
- **Cost efficiency** â€” Minimized or zero cloud egress fees, pay only for your own platform usage

Nexalis Cloud follows a **medallion architecture**:

1. **Bronze Layer** â€” Raw sensor data ingested from Nexalis Agents
2. **Silver Layer** â€” Cleansed and standardized time-series data
3. **Gold Layer** â€” Aggregated, enriched, and ready-to-analyze datasets

You receive access to the **Gold Layer** tables:
- `gold_<client>_hf_std` â€” High-frequency standardized data (raw sensor readings)
- `gold_<client>_avg_std` â€” Time-averaged data with statistics (mean, std dev, min, max)
